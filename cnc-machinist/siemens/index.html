Siemens Turning & Setup Toolkit

Welcome to the Sinumerik G-Code Suite

Siemens Sinumerik controls (including the 840D, 828D, and 808D) utilize an extremely powerful, high-level CNC programming language. Unlike ISO standard G-code (Fanuc/Haas), Siemens supports named variables, parameterizable canned cycles, local contour labels, and advanced tool management. This suite provides professional shop-floor tools to write correct Siemens turning code and generate standardized offset setup cards.

G-Code Turning Fundamentals

Turning on Siemens systems is programmed in the G18 Z-X plane. Tool coordinate axes are defined as:

Command Description
G18 Select ZX Plane (Default for Lathe turning)
G95 Feed per Revolution (e.g., F0.2 = 0.2mm/rev)
G96 Constant Cutting Speed active (S is Surface m/min)
LIMS Spindle RPM Limit during G96 (e.g., LIMS=3000)
G97 Constant Spindle Speed active (S is Spindle RPM)

Cycles Sinumerik Lathe Cycles

Siemens uses built-in cycle parameters to automate complex operations instead of long, manual coordinates:

Cycle Operation
CYCLE95 Stock Removal / Contour Roughing & Finishing
CYCLE93 Grooving & Part-off Cycle
CYCLE97 Thread Cutting (Single/Multi-start)
CYCLE98 Thread Chain (Continuous threads)
CYCLE99 Simple Threading Cycle

T & D Tool Names & Cutting Edges

In Siemens controls, tools are selected by their alphanumeric name or number, and the active cutting edge offset is designated by a D-number:

  • T="ROUGH_80" D1: Call tool ROUGH_80 and activate its 1st cutting edge offsets.
  • Tool Radius Compensation is activated with G41 (left of path) or G42 (right of path), and cancelled with G40.
  • The tool point orientation (direction 1-9) must be accurately defined in the offsets page for radius compensation to calculate correctly.

CYCLE95 Contour Programming

Siemens contour cycles can reference an internal block range using starting and ending labels, making separate subprogram files unnecessary:

; Call cycle95 roughing
CYCLE95("CONTOUR1:CONTOUR1_END", 2.0, 0.2, 0.1, 0, 0.2, 0.15, 1, 1)
